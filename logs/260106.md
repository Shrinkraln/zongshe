## 1. 核心问题复盘

### 问题一：`requirements.txt` 文件丢失
*   **现象**: 拉取官方组件后，执行 `pip install -r .../requirements.txt` 报错文件不存在。
*   **原因**: micro-ROS 官方在 Humble 分支更新中移除了该文件。
*   **解决方案**: 手动安装依赖包，**必须安装到 ESP-IDF 的虚拟环境中**。
    ```bash
    # 注意：必须使用 IDF 环境下的 python 路径，或者先激活 idf 环境
    . $HOME/esp/esp-idf/export.sh
    pip install catkin_pkg lark-parser empy==3.3.4
    ```

### 问题二：编译时 `catkin_pkg` 找不到
*   **现象**: 报错 `ModuleNotFoundError: No module named 'catkin_pkg'`。
*   **原因**: 依赖包安装到了系统 Python (/usr/bin/python)，而 IDF 编译使用的是 `~/.espressif/python_env/...` 下的 Python。
*   **解决方案**: 显式指定 IDF Python 解释器安装：
    ```bash
    /home/用户名/.espressif/python_env/idf5.1_py3.10_env/bin/python -m pip install catkin_pkg lark-parser
    ```

### 问题三：`ros2_tracing` 网络超时 (最顽固的坑)
*   **现象**: `idf.py build` 下载源码时卡死，或报错 `Failed to connect to gitlab.com`，导致 `rcl` 库缺少依赖 `tracetools` 无法编译。
*   **原因**: micro-ROS 自动下载脚本依赖 GitLab，国内访问极不稳定。
*   **解决方案**: 手动替换为 GitHub 镜像。
    1. 进入源码目录: `cd components/micro_ros_espidf_component/micro_ros_src/src`
    2. 删除坏目录: `rm -rf ros2_tracing`
    3. 手动克隆: `git clone -b humble https://github.com/ros2/ros2_tracing.git`
    4. 忽略测试(防止滚雪球报错): 
       `touch ros2_tracing/tracetools_test/COLCON_IGNORE`
       `touch ros2_tracing/test_tracetools/COLCON_IGNORE`

### 问题四：`fatal error: rcl/rcl.h: No such file`
*   **现象**: micro-ROS 库似乎编译了，但主程序找不到头文件。
*   **原因**: 之前的编译因为报错中断，导致最后一步“搬运头文件到 include 目录”的脚本没有执行。后续编译因为缓存存在，以为工作已完成，不再搬运。
*   **解决方案**: **彻底清理并重构**。
    ```bash
    # 删除组件内的编译缓存 (触发重新打包)
    rm -rf components/micro_ros_espidf_component/micro_ros_src/build
    rm -rf components/micro_ros_espidf_component/micro_ros_src/install
    # 删除可能错误的 include
    rm -rf components/micro_ros_espidf_component/include/*
    # 重新编译
    idf.py build
    ```

## 2. 以后创建新项目的“黄金流程” (Fast Path)

为了避免重复下载和编译 micro-ROS 静态库（非常耗时且易出错），建议采用 **“复制组件法”**。

### 步骤 1：准备组件
找到之前成功编译过的项目（例如 `micro_ros2`），将整个 `components` 文件夹复制到新项目。
*   **优势**: 里面的 `micro_ros_src` 已经包含了修好的 `ros2_tracing`，无需联网下载，直接编译即可。

### 步骤 2：创建新项目
```bash
# 1. 创建空项目
idf.py create-project my_new_robot
cd my_new_robot

# 2. 复制这一步做好的组件过来 (假设旧项目在 ../micro_ros2)
cp -r ../micro_ros2/components .

# 3. 修改 main/CMakeLists.txt 注册组件
# 内容:
# idf_component_register(SRCS "main.c"
#                     INCLUDE_DIRS "."
#                     PRIV_REQUIRES micro_ros_espidf_component)
```
