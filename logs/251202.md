1. espidf版本不匹配，需要是4.4.5或者5.5
2. ros2 run micro_ros_setup 创建工作区是使用自己的python虚拟环境，需要在此环境中安装依赖
```bash
#创建工作区，配置，构建，烧录都在此进行
source /root/.espressif/python_env/idf4.1_py3.10_env/bin/activate
```
3. 清除旧版本指向干扰
```bash
#清理旧版本以及环境缓存
rm -rf /root/.espressif
rm -rf /ros2_ws/firmware/toolchain/esp-idf
#重新安装指定的版本
cd /ros2_ws/firmware/toolchain
git clone -b v5.2.2 --recursive https://github.com/espressif/esp-idf.git
#初始化
cd esp-idf_v5.2
git submodule update --init --recursive
#创建激活虚拟环境
python3 -m venv ~/.espressif/python_env/idf5.2_py3.10_env
source ~/.espressif/python_env/idf5.2_py3.10_env/bin/activate
#安装依赖
python -m pip install --upgrade pip setuptools
./install.sh
```
5. 新的docker创建指令，已经添加到dockerfile中
```bash
docker run -it --privileged \
  --name ros_espidf_dev \
  -v ~/ros2_ws:/ros2_ws \
  -v /dev:/dev \
  --network host \
  -e http_proxy=http://127.0.0.1:7890 \
  -e https_proxy=http://127.0.0.1:7890 \
  ros_espidf_dev
```
6. 创建工作区的时候需要安装依赖，docker镜像源没有此依赖，可能需下载完整依赖
```bash
apt-get update && apt-get install -y \
build-essential git cmake ninja-build ccache \
python3-dev python3-pip python3-setuptools python3-venv \
libncurses-dev libssl-dev libffi-dev libasio-dev libtinyxml2-dev \
libboost-all-dev libcurl4-openssl-dev
python -m pip install -U \
    catkin_pkg empy lark-parser pyparsing pyyaml
```
