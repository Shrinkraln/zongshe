# TurtleBot4 Gazebo 赛道仿真 — LiDAR 射线飘移问题排查与解决

## 1. 问题描述

在自定义 Gazebo Ignition 赛道世界中，TurtleBot4 的 LiDAR 扫描出现以下异常：

- **射线穿透**：部分激光射线直接穿透墙体，无法正确检测障碍物
- **射线缺失**：特定方向的射线完全消失，RViz 中显示为空白区域
- **尖锐突刺**：个别射线返回极近距离值，在点云中形成尖刺
- **密度不均**：射线分布疏密不一，部分区域密集，部分区域稀疏
- **随旋转变化**：异常区域相对机器人固定，但相对世界随旋转变化

该问题在官方 maze.sdf 世界中不存在，仅在自定义赛道世界中出现。

## 2. 排查过程

### 2.1 排除传感器噪声/墙面材质（×）

最初怀疑墙面材质导致激光反射异常。但由于是 Gazebo 仿真环境且墙体材质一致，排除此可能。

### 2.2 排除机器人自身遮挡（×）

通过 Gazebo 激光可视化检查了 TurtleBot4 的 URDF 模型（shell_link 的 collision mesh、rplidar_link 的碰撞盒、oakd_camera_bracket 等部件）。但同一机器人在官方 maze 中正常，排除此可能。

### 2.3 墙体高度不足（部分原因）

发现赛道墙壁高度仅 0.2m，而 LiDAR 安装在 base_link 上方约 0.193m，几乎与墙顶齐平。通过 SDF 中设置 `<scale>1 1 2</scale>` 将墙高加倍到 0.4m，问题有所改善但未完全解决。

### 2.4 Mesh 法线/封盖问题（×）

给 DAE mesh 加上底面封盖，使墙体成为封闭几何体。修改后问题依然存在，证明问题不在于 mesh 封闭性。

### 2.5 Box Collision 但结构错误（×）

将 collision 从 mesh 改为 box 拼接，但把所有 box 放在同一个 link 内。问题依旧存在。

### 2.6 对比官方 maze.sdf 找到根本原因（✓）

对比官方 maze.sdf 发现关键结构差异：

| | maze.sdf（正常） | 自定义赛道（异常） |
|---|---|---|
| collision 类型 | box 基本几何体 | mesh 或单 link 内多 collision |
| model/link 结构 | 1 个 model, 多个 link | 多个 model 或单 link |
| 每个 link 内容 | 1 个 collision + 1 个 visual | 几百个 collision |
| Sensors 插件 | 无（由 launch 加载） | 有（导致 GUI 闪退） |
| ground_plane | 有 `__default__` 插件 | 无 |

## 3. 根本原因

| 因素 | 说明 |
|---|---|
| Mesh Collision | Ignition Gazebo 的射线检测（ray casting）对复杂 mesh 的穿透/反射计算不可靠，尤其是非封闭或薄壁 mesh |
| SDF 结构不当 | 单个 link 内放置大量 collision 对象，Ignition 物理引擎处理异常 |
| 墙体高度不足 | 原始墙高 0.2m 与 LiDAR 安装高度 0.193m 几乎齐平，射线擦过墙顶 |
| Sensors 插件冲突 | SDF 中包含 Sensors 插件与 TurtleBot4 launch 重复加载，导致 GUI 闪退 |

## 4. 最终解决方案

采用与官方 maze.sdf 完全一致的结构重新生成赛道 SDF：

### 4.1 纯 Box 拼接替代 Mesh

将 SVG 路径上的每两个相邻点生成一个独立的 box，通过 pose 中的 yaw 角度让 box 沿路径方向摆放。相邻 box 重叠 2cm 消除缝隙。

### 4.2 每个 Box 是独立 Link

与 maze.sdf 一致，每个 box 作为独立的 link 放在同一个 model 下。每个 link 包含一个 collision 和一个 visual，collision 和 visual 使用相同的 box 几何体。

### 4.3 不包含 Sensors 插件

SDF world 文件中不包含 Sensors 插件，该插件由 TurtleBot4 的 launch 文件在 spawn 机器人时自动加载。重复声明会导致 Gazebo GUI 闪退。

### 4.4 墙体参数

墙高 0.4m（远高于 LiDAR 安装高度 0.193m），墙厚 0.05m。

## 5. 经验总结

| 经验 | 详细说明 |
|---|---|
| 避免用 mesh 做 collision | Ignition Gazebo 的 ray casting 对复杂 mesh 不可靠，始终优先使用 box/cylinder 等基本几何体 |
| 参考官方 SDF 结构 | 新建世界时先研究官方示例（如 maze.sdf）的 model/link 组织方式 |
| 一个 link 一个 collision | 不要在单个 link 内放置大量 collision 对象，应该每个 collision 放在独立 link 中 |
| 不要重复 Sensors 插件 | TurtleBot4 launch 会自动加载，world SDF 中不应包含，否则导致 GUI 崩溃 |
| 墙高 > LiDAR 高度 | LiDAR 安装高度约 0.193m，墙高应至少 0.35m 以上避免射线擦过墙顶 |
| Box 重叠消除缝隙 | 相邻 box 重叠 1-2cm，确保弯道处无缝隙让射线穿透 |
| 部署脚本的作用 | 部署脚本通过重写 SDF 去除 Sensors 插件解决 GUI 闪退问题，并非多余 |

# Camera 问题解决
那问题就在于 RViz2 的 Camera 显示插件本身的渲染方式。Camera 插件会把图像作为背景投影到 3D 视角中，它看起来"扭曲"是因为它在做 3D 透视叠加，而不是简单地显示 2D 图像。你之前看到的那些蓝色、红色其实是 Gazebo 场景中的物体（激光扫描线、地图等）叠加在相机画面上的效果。要使用image插件作为2D展示效果

```bash
#保存配置
sudo cp user_0.rviz /opt/ros/humble/share/turtlebot4_viz/rviz/user0.rviz
```
