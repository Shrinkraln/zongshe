### 1. 为什么要用这个算法？（核心逻辑）

IMU 里的两个传感器都有“硬伤”，单独使用谁都不行：

1.  **陀螺仪 (Gyroscope)**：
    *   **优点**：反应极快，能精准测量旋转的**角速度**。
    *   **缺点**：会有**漂移**。你通过积分（速度×时间=角度）计算角度时，微小的误差会不断累积。时间久了，明明小车没动，算出来的角度还会积累。
2.  **加速度计 (Accelerometer)**：
    *   **优点**：静止时能精准测量**重力方向**（也就是哪边是下），没有漂移，长期来看很准。
    *   **缺点**：**噪声大**，且对震动敏感。小车一晃动，加速度计的数据就会乱跳，没法用来算瞬时姿态。

**Mahony 算法的思路（互补滤波）**：
> “平时我主要**信陀螺仪**（因为它快），通过积分算姿态。但我知道陀螺仪会飘，所以我每隔一小段时间（比如 10ms），就**拿加速度计测出的重力方向**来校准一下陀螺仪，把飘出去的误差拉回来。”

### 2. 关键参数详解 (Parameters)

在 `imu_adapter.c` 中，最重要的两个参数是 `Kp` 和 `Ki`：

#### `Kp` (比例增益 - Proportional Gain)
*   **代码定义**：`#define Kp 10.0f`
*   **含义**：**“我有多信任加速度计？”**
    *   这个参数决定了加速度计对陀螺仪漂移的**修正力度**。
*   **调参指南**：
    *   **Kp 太小**：修正太慢。如果陀螺仪漂移了，姿态数据需要很长时间才能慢慢回正。
    *   **Kp 太大**：修正过头。虽然消除了漂移，但加速度计的噪声（震动）会被引入，导致姿态数据在静止时也会“抖动”。
    *   **推荐**：对于小车，`2.0f` 到 `10.0f` 之间通常比较合适。

#### `Ki` (积分增益 - Integral Gain)
*   **代码定义**：`#define Ki 0.005f`
*   **含义**：**“由于器件老化或温度变化产生的持续偏差（零点漂移）有多少？”**
    *   陀螺仪即使静止不动，读数可能也是 0.01 而不是 0。`Ki` 用来慢慢消除这个固定的偏差。
*   **调参指南**：
    *   通常设得很小（如 `0.005` 或 `0.0f`）。
    *   如果你发现小车静止放了很久，姿态还在以一个极其缓慢的速度一直转圈，可以稍微加大一点 `Ki`。

#### `sampleFreq` / `dt` (采样频率/时间间隔)
*   **代码逻辑**：`float dt = ...` (通过 `esp_timer` 计算)
*   **含义**：积分的时间基准。
*   **重要性**：**非常关键！** 如果你的 `dt` 算错了（比如代码里写死 0.01s，实际运行间隔是 0.02s），那么算出来的角度就会慢一倍或快一倍。我们代码里用了实时计算 `dt`，这是最稳妥的。

### 3. 算法内部步骤拆解 (Magic Inside)

函数 `MahonyAHRSupdateIMU` 到底干了什么？

#### 第一步：归一化 (Normalization)
```c
recipNorm = invSqrt(ax * ax + ay * ay + az * az);
ax *= recipNorm; ...
```
*   **含义**：把加速度计读到的向量长度缩放为 1。因为我们只关心重力的**方向**，不关心大小（不管是 1g 还是 2g，向下的方向是一样的）。
*   *注：`invSqrt` 是著名的“快速平方根倒数算法”，出自雷神之锤游戏，计算速度极快，适合嵌入式。*

#### 第二步：推算“理论重力方向”
```c
halfvx = q1 * q3 - q0 * q2; ...
```
*   **含义**：根据当前算出来的姿态（四元数），推算一下：“如果我现在是这个姿态，那么重力应该指向哪里？”

#### 第三步：计算误差 (Error Calculation)
```c
halfex = (ay * halfvz - az * halfvy); ...
```
*   **含义**：**叉乘 (Cross Product)**。
    *   向量 A：加速度计**实际测到**的重力方向。
    *   向量 B：通过姿态**推算出**的重力方向。
    *   A 和 B 的叉乘结果，就是这两个方向的**偏差**（误差向量）。这个误差告诉我们要把姿态往哪个方向“掰”才能对准重力。

#### 第四步：反馈修正 (Feedback)
```c
gx += Kp * halfex; ...
```
*   **含义**：把刚才算出来的误差，乘上 `Kp`，直接加到陀螺仪的数据上。
    *   这就相当于：虽然陀螺仪说没转，但加速度计发现姿态歪了，我就**人为地给陀螺仪加一个旋转速度**，让姿态快速转回到正确位置。

#### 第五步：姿态更新 (Integration)
```c
q0 += (-qb * gx - qc * gy - q3 * gz); ...
```
*   **含义**：利用修正后的角速度，通过积分更新四元数 `q0, q1, q2, q3`。

### 4. 输出结果：四元数 (Quaternion)

最后输出的 `q0(w), q1(x), q2(y), q3(z)` 是什么？

*   **定义**：四元数是一种数学工具，用来无死角地表示 3D 旋转。
*   **为什么要用它（而不是欧拉角）？**
    *   **欧拉角 (Roll, Pitch, Yaw)**：容易理解（翻滚、俯仰、偏航），但在垂直向上（90度）时会发生**万向节死锁 (Gimbal Lock)**，导致数学计算崩溃。
    *   **四元数**：虽然人类难以直观理解这 4 个数字，但计算机处理起来非常丝滑，**永远不会死锁**。
*   **ROS 2 标准**：ROS 2 的 `sensor_msgs/Imu` 消息强制要求使用四元数。你的网页端 JS 代码或者 ROS 里的 RViz 工具，会自动把这 4 个数字转换成可视化的箭头或角度。
