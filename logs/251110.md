1. esp32-s3的USB是内置，使用ls -l /dev/ttyACM0*查看。而节点一旦创建之后的名字是不会发生变化的，但是芯片有各自的序列号，在宿主进行指令
```bash
udevadm info -a -n /dev/ttyACM0 | grep -E "serial|idVendor|idProduct"
```
2. 如果你主机开了代表（VPN / Clash），容器内不会自动跟随
你必须显示告诉 docker：
```bash
docker run -it --privileged \
  --name ros_espidf_dev \
  -v ~/ros2_ws:/ros2_ws \
  -v ~/esp/esp-idf_v5.2:/esp/esp-idf_v5.2 \
  -v /dev:/dev \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  --network host \
  -e http_proxy=http://127.0.0.1:7890 \
  -e https_proxy=http://127.0.0.1:7890 \
  ros_espidf_dev
  
#替换 7890 为你的代理端口
#如果你用 Clash，通常就是 7890。
#这样 GitHub 的下载就不会崩了。
```
3. 更正了mcu和pc的结构关系
   1. ros2_ws/src/下放pc包
   2. ros2_ws/firmware才是mcu烧录的工具包，ros2_ws/firmware/dev_ws/src/放mcu包
4. python-nose不适合，替换为python-test
```bash
#在容器内创建文件：
nano /etc/ros/rosdep/sources.list.d/01-custom-nose.list
#添加：
yaml file:///etc/ros/python3-nose-fix.yaml
#再创建映射文件：
nano /etc/ros/python3-nose-fix.yaml
#填写：
python3-nose:
  ubuntu: [python3-pytest]
#更新 rosdep：
rosdep update
```
5. ros2 pkg create --build-type ament_python mcu_py_pkg创建包mcu_py_pkg
6. 不能挂载主机的工具连，会指向错误
7. 正确的结构
```bash
/ros2_ws
├── src
│   ├── pc_pkg          # PC 端 ROS2 节点包
│   └── test_agent      # 可能是 micro-ROS 测试程序
├── firmware            # MCU 固件相关
│   ├── freertos_apps   # MCU 端 freertos 示例包
│   └── dev_ws          # MCU 自己的 colcon 工作区
├── build
├── install
└── log
```
